### CROSS-CUTTING PATTERN — mobile Sort & Filter drawer — BUILT 7/23 (49ceb56, 299cff0)
Status: **live on the preview branch.** Reusable pieces: `renderFilterDrawerTrigger(id)`
+ `filterDrawerHead()` + shared controller in `renderCustomSelectScript` (scrim, Esc,
body-lock, focus). Wired on **Tour Stats** (6 controls) and **Nick** (view/type/sort).
Song Index left alone deliberately — 3 light controls, already clean in the audit.
Known trade: the sticky header fades while a drawer is open (stacking-context trap;
drawer owns dismissal). Original spec follows.
Alex: "Do we need a sort and filter button and then slide in a sidebar that lets people
do that for mobile? That's probably the ticket... right now there are like 5 filters
above each so it's not mobile optimization." — and it repeats across pages.
- **Build one reusable mobile control:** a single **Sort & Filter** button that opens a
  **slide-in drawer/sheet** holding the sort + all filters. Under ~600px, collapse the
  inline control wall into this one button. Desktop keeps the inline controls.
- **Apply everywhere a filter/sort wall exists:** Tour Stats (All/Originals/Covers +
  Not-played + Rarity + Find + Sort = 5+ controls), Nick (VIEW + chips + SORT), Song
  Index, and any other filtered list. One component, reused.
- This single pattern resolves the "stats are a mess / 5 filters stacked" complaint on
  the homepage AND the same problem on subpages in one build.

### NICK SECTION — data-model spec (Alex, 2026-07-23) — REBUILD, not a tweak
Strip to what people actually want:
1. **How many / which songs Nick HAS played.**
2. **Which in-rotation songs he HASN'T played yet.**
3. **Optional "likelihood"** — a composite of (a) avg plays per 10 shows, (b) last-time-
   played, (c) frequency over the last 100 shows, **excluding bustout anomalies** (his
   example: "Jack Straw" — a one-off must not read as likely). Plain-language.
   **Never "Heat"** ("HEAT means jack shit"). The 0-100 number is already off the preview
   (commit eba4f4f); the underlying `data-heat`/sort must be renamed/reframed to
   "likelihood" if kept, or dropped.
- **Cut redundancy:** the section stacks a rig poster + a 50% tri-bar
  (overall/originals/covers) + THREE controls doing the same filter job (VIEW dropdown,
  All/Originals/Covers chips, SORT dropdown). Collapse to the one drawer above + keep
  the two core reads (played / not-yet-played).
- Sticky section-nav also bleeds over the stat tiles on scroll (same overlap as §2).
