# Mobile QA — burnthday.com (live) — started 2026-07-23

Working scaffold for the mobile QA session. Findings from a 375×812 pass of the
**live** site (real Chrome, iPhone-class viewport). Verdict up top, then per-issue
detail with the exact element + fix direction, then the not-yet-checked queue.

Method: DOM overflow probe (element rects vs viewport, skipping anything an
ancestor clips) + section-by-section screenshots. So "overflow" here means a real
draggable horizontal scroll, not a masked marquee.

---

## VERDICT: not fully optimized. Homepage has 2 real bugs + 2 cosmetic. Subpages mostly clean.

The homepage is the weak surface (it's 24,110px tall on mobile and by far the most
complex). Subpages checked so far are good — the Song Index is actually the model
for how a data table should behave on mobile.

**Do NOT fix on the homepage without Alex's word** — he has the homepage locked.
Everything below the homepage line is fair game. Homepage items are logged for his call.

## HOMEPAGE findings

### 1. Horizontal overflow ~10px — HIGH (causes the sideways jiggle)
Body scrolls to **385px in a 375px viewport**. Two unclipped offenders:
- `nav.home-nav` — width 385, right edge 385. The fixed section-nav ("SONG
  POSSIBILITIES › TOUR STATS › 2026 SETLISTS").
- `div.sheet-scrawl` — width 395, right edge 385. The handwritten "LOS ANGELES CA
  1 2 3 4" scrawl across the top of the white song sheet.

Fix direction: both need to respect the 375 gutter (max-width / clamp the scrawl
to the sheet width; constrain the nav to the content column). The prior "375px
technical sweep" in the handoff claimed no sideways scroll — it regressed or was
measured on a different build. Re-verify after fixing.

### 2. Sticky section-nav overlaps section headings — HIGH (text on text)
The fixed `home-nav` bar sits on top of the section heading whenever a section
lands at the top of the viewport. Observed colliding with:
- board-intro headline ("Widespread Panic song possibilities…")
- "Tour stats follows the songs on…" (top line hidden behind the nav)
- "Nick Johnson has played just over half…" (top line hidden)
- "2026 setlists" (heading top clipped)

It reads as two layers of text in the same band. Fix direction: scroll-margin-top
on section headings equal to the nav height, or don't let the nav paint over the
heading row. This is the single most-visible mobile problem.

### 3. Tour Stats table — cramped, inconsistent truncation — MEDIUM
The homepage Tour Stats table goes horizontal-scroll on mobile (acceptable
pattern) but:
- Song names are inconsistent: "Drums" truncates to "Dru…" while "Little By
  Little" and "Ride Me High" wrap to 3 lines each.
- Column headers cut mid-word: "Pla" (Plays), "Last / us" (Last / usual gap),
  "Last" (Last played).

Contrast: the **Song Index subpage** (`/songs/`) solves the same problem correctly
with a stacked-card layout (title / status badge / plays), no horizontal scroll,
no truncation. That's the pattern this table should adopt on mobile.

### 4. Nick stat tiles — label crowding — LOW/MEDIUM
"531 SONG PLAYS" and "124 IN THE WOODSHED" cram the label beside the big number
(labels wrap to 2–3 lines pinned to the number's right). "27 SHOWS" is clean
because its label is short. Fix direction: stack label under number on mobile like
the Tour Stats tiles (29 / SHOWS PLAYED) do.

### 5. Rarity ticker clipped both edges — LOW (pre-existing, by design)
The hero rarity ticker ("…ULTRA RARE Contentment Blues · ★★ ULTRA RARE Fla…")
is edge-masked and reads as cut-off text on a static glance. It's a marquee that
crawls, and the mask matches desktop — not a mobile regression. Left here so the
QA session knows it's known, not new.
